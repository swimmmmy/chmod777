# GNSS 미코닝(Meaconing) 및 Pseudolite 기반 PNT 기만 시나리오

# 최종 주제

**"GNSS 신호 물리적 시간 지연(Meaconing)을 통한 Timing(T) 정보 교란이 PNT(Position, Navigation, Timing)에 미치는 영향 분석"**

> 공격자는 시스템을 해킹하거나 내부 네트워크에 침투하지 않는다. 단지 정상적인 GNSS 신호를 매우 짧은 시간(나노초 단위) 지연시켜 재전송할 뿐이다. 그러나 수신기는 이를 정상 신호로 인식할 수 있으며, 위치(Position), 항법(Navigation), 시각(Timing)에 오류가 발생해 PNT 기반 무기체계의 운용에 영향을 줄 수 있다.

# 알아놔야 할 개념

## PNT란?

**P**ositioning(위치) + **N**avigation(항법) + **T**iming(시각)

현대 군사 시스템은 대부분 PNT 정보를 기반으로 운용된다.

* 드론 및 무인기 비행
* 군함·잠수함 항법
* 유도무기 표적 추적
* 군 통신망 시간 동기화
* 지휘통제(C2) 시스템

즉, PNT 정보의 신뢰성은 현대 군 작전 수행의 핵심 요소이다.
특히 Timing(T)은 Position과 Navigation 계산의 기준이 되는 정보로, 매우 작은 시간 오차도 전체 PNT 정확도에 영향을 줄 수 있다.

---

## GNSS란?

GNSS(Global Navigation Satellite System)는 위성을 이용하여 PNT 정보를 제공하는 위성항법 시스템의 총칭이다.

| 국가  | 시스템     | 위성 수(약) |
| --- | ------- | ------- |
| 미국  | GPS     | 31      |
| 러시아 | GLONASS | 24      |
| 유럽  | Galileo | 27      |
| 중국  | BeiDou  | 35      |

---

## GNSS 신호의 특징

GPS 위성은 약 20,000km 이상의 고도에서 신호를 송신한다.

* 위성 송신 출력 : 약 **25W**
* 지상 수신 전력 : 약 **-160 dBW**

지상에서는 매우 약한 신호만 수신되므로 다양한 전파 간섭이나 신호 기만 공격에 상대적으로 취약하다.

---

# 핵심 공격 기법

## 1. 미코닝(Meaconing)

> 정상적인 GNSS 신호를 수신한 뒤, 내용은 변경하지 않고 시간만 아주 짧게 지연시켜 다시 송신하는 공격 기법

| 구분     | 재밍    | 스푸핑      | 미코닝            |
| ------ | ----- | -------- | -------------- |
| 방식     | 신호 방해 | 가짜 신호 생성 | 정상 신호 지연 후 재송신 |
| 데이터    | 없음    | 가짜       | 정상 데이터 그대로     |
| 탐지 난이도 | 쉬움    | 보통       | 상대적으로 어려움      |

### 미코닝이 탐지하기 어려운 이유

* 신호 내용이 정상 데이터와 동일하다.
* 자연적인 전파 지연과 구별하기 쉽지 않다.
* 수신기는 일반적으로 더 안정적이고 강한 신호를 선택할 수 있다.
* 기존 GNSS 수신기에서는 이상 여부를 즉시 판단하기 어렵다.

---

## 2. Pseudolite

Pseudolite(Pseudo + Satellite)는 지상 또는 드론·항공기 등에 설치되어 GNSS 위성처럼 신호를 송신하는 장치이다.

GPS 위성보다 훨씬 가까운 위치에서 신호를 송신하므로 수신기에는 상대적으로 강한 신호로 인식될 수 있다.

Pseudolite 자체는 하나의 장치(플랫폼)이며, 여기에 다양한 GNSS 신호 생성 또는 기만 기법을 적용할 수 있다.

---

## 3. 두 기법의 결합 시나리오

```
드론 또는 이동 플랫폼

        ↓

Pseudolite 장비 운용

        ↓

Meaconing 방식의 지연 신호 송신

        ↓

GNSS 수신기 정상 신호로 인식

        ↓

PNT 정보 왜곡
```

---

# 공격 시나리오

## 가정

적 세력이 운용하는 이동 플랫폼(드론, 항공기, 차량 등)에 Pseudolite 장비를 탑재하여 GNSS 환경에 지연된 정상 신호를 송신하는 상황을 가정한다.

---

## 단계별 흐름

```
① GNSS 수신 환경에 지연 신호 유입

          ↓

② 수신기가 정상 GNSS 신호로 판단

          ↓

③ Timing(T) 정보에 미세한 오차 발생

          ↓

④ ΔR = c × Δt
   (거리 계산 오류 발생)

          ↓

⑤ Position(Positioning) 및 Navigation 정보 오류 발생

          ↓

⑥ PNT 기반 시스템의 연쇄적인 운용 오류 발생
```
ΔR = c × Δt
```

* ΔR : 거리 오차
* c : 빛의 속도
* Δt : 시간 오차

예를 들어,

* 1 ns → 약 30 cm
* 10 ns → 약 3 m
* 100 ns → 약 30 m

즉, GNSS에서 Position과 Navigation의 정확도는 Timing 정보의 정확성에 직접적으로 의존한다. 본 시나리오는 작은 Timing 오차가 PNT 전체에 어떤 영향을 미치는지를 확인하는 데 목적이 있다.

# 예상 영향

| 대상    | 예상 영향              |
| ----- | ------------------ |
| 군용 드론 | 위치 오차로 임무 수행 오류 가능 |
| 항공기   | 항법 오차 및 접근 절차 영향   |
| 군함    | 항로 이탈 가능성          |
| 유도무기  | 표적 좌표 오차 발생 가능     |
| 군 통신망 | 시간 동기화 오류 가능       |
| 암호체계  | 시간 기반 인증 실패 가능     |

※ 실제 영향은 운용 환경, 복합항법(INS), 보안 기능 적용 여부 등에 따라 달라질 수 있다.

# 실증 방법

## Pix4 SITL 기반 GNSS 스푸핑 시뮬레이션

실제 RF 송신 장비를 사용하지 않고 시뮬레이션 환경에서 GNSS 기만 상황을 재현한다.

### 사용 도구

* GPS-SDR-SIM
* Pix4 SITL
* GCS(Ground Control Station)

### 실험 절차

### 1단계 — GNSS 위치 데이터 생성

* GPS-SDR-SIM을 이용하여 가상의 GNSS 위치 데이터를 생성
* 예시

  * 실제 위치 : 청주
  * 가상 위치 : 논산
* 생성된 위치 정보를 시뮬레이션 환경에서 사용할 수 있도록 준비

---

### 2단계 — Pix4 SITL 환경 적용

* Pix4 SITL 시뮬레이터 실행 (팀 보유 환경)
* 생성한 가상 GNSS 위치 정보를 시뮬레이션에 적용
* 드론은 입력된 GNSS 정보를 기준으로 자신의 위치를 판단

---

### 3단계 — GCS 화면 확인

* GCS(Ground Control Station)에서 드론의 위치 정보를 확인
* 정상 환경과 기만 환경의 화면을 비교
* 위치 정보가 어떻게 달라지는지 캡처 및 영상으로 기록

---

| 정상 환경         | 기만 환경              |
| ------------- | ------------------ |
| 정상 GNSS 위치 정보 | 가상의 GNSS 위치 정보 적용  |
| 드론 위치 정상 표시   | 변경된 위치로 인식         |
| 정상적인 항법 수행    | 위치 정보 왜곡에 따른 항법 변화 |


> **운용자는 GCS 화면상에서 정상적으로 위치 정보를 확인하고 있다고 판단할 수 있지만, 드론은 실제 위치와 다른 GNSS 정보를 기준으로 동작할 수 있다.**
